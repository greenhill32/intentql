The left side is what breaks AI browsers today.

Every website redesign. Every A/B test. Every dynamic load. The agent parses HTML, guesses at selectors, retries when it fails. This is how Opera Neon, browser copilots, and every automation tool works right now.

It doesn't scale. It can't.

Agents don't need better eyes.
They need an API.

IntentQL is a simple proposal: websites publish an /agent.json file declaring what they can do. Agents read the contract and call stable endpoints. The UI becomes irrelevant to automation.

Three moving parts

The contract

/agent.json

A machine-readable file declaring what the site can do. Intents, endpoints, parameters, response shapes.

GET /agent.json
The endpoints

Stable APIs

Normal REST endpoints that do the work. Search products, check stock, add to cart. No DOM required.

POST /api/cart/add
The UI

For humans only

The website looks however you want. Redesign freely. Agents don't care — they're not looking at it.

Everyone wins differently

Agents

Site owners

Control what agents can do. Distinguish good actors from scrapers. Redesign your UI without breaking integrations.

Enterprise

Contracts, not heuristics. Legal clarity on what's permitted. Audit trails. The kind of guarantees procurement requires.
